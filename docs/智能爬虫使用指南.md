# Boss直聘智能爬虫使用指南

## 🎯 核心特性

### 1. **登录态复用**
- ✅ 首次登录后自动保存Cookie
- ✅ 后续运行自动使用已保存的Cookie
- ✅ 无需每次都手动登录
- ✅ Cookie文件位置: `data/cookies.json`

### 2. **智能爬取控制**
- ✅ 精确控制每个关键词爬取数量
- ✅ 设置总计最大爬取数量
- ✅ 自动分页，无需手动控制
- ✅ 达到限制自动停止

### 3. **反爬虫对策**
- ✅ 随机延时（3-6秒）
- ✅ 智能滚动页面（模拟真人）
- ✅ 隐藏自动化特征
- ✅ 随机User-Agent
- ✅ 检测并处理验证码

### 4. **数据安全**
- ✅ 中断保存（Ctrl+C会保存已爬数据）
- ✅ 异常保存（出错也会保存数据）
- ✅ Cookie加密存储
- ✅ 完整的日志记录

## 📝 快速开始

### 方式1：使用默认配置（推荐新手）

```bash
python run_smart_crawler.py
```

**默认配置**:
- 城市: 北京、上海
- 关键词: Python
- 每个关键词: 50条
- 总计: 200条

### 方式2：自定义配置

```bash
python run_smart_crawler.py \
  --cities 北京 上海 深圳 杭州 \
  --keywords Python Java 数据分析 \
  --per-keyword 30 \
  --total 150
```

### 方式3：无头模式（后台运行）

```bash
python run_smart_crawler.py --headless
```

## 🔧 参数说明

| 参数 | 说明 | 默认值 | 示例 |
|-----|------|--------|------|
| `--cities` | 城市列表 | 北京 上海 | `--cities 北京 深圳` |
| `--keywords` | 关键词列表 | Python | `--keywords Python Java` |
| `--per-keyword` | 每个关键词最大数 | 50 | `--per-keyword 30` |
| `--total` | 总计最大数 | 200 | `--total 100` |
| `--headless` | 无头模式 | False | `--headless` |
| `--cookie-file` | Cookie文件路径 | data/cookies.json | `--cookie-file cookies.json` |

## 📖 使用流程

### 第一次使用

1. **启动爬虫**
   ```bash
   python run_smart_crawler.py
   ```

2. **浏览器会自动打开**
   - 等待几秒加载

3. **手动登录Boss直聘**
   - 点击页面的"登录"按钮
   - 扫码或账号密码登录
   - 登录成功后回到终端

4. **按回车继续**
   - 爬虫自动开始工作
   - Cookie会自动保存

5. **等待完成**
   - 自动爬取数据
   - 显示进度信息
   - 数据自动保存

### 第二次及以后使用

1. **直接启动**
   ```bash
   python run_smart_crawler.py
   ```

2. **自动登录**
   - 使用保存的Cookie
   - 无需手动登录
   - 直接开始爬取

## ⚙️ 爬取控制参数

### 内置控制（在代码中）

```python
self.config = {
    'delay_range': (3, 6),          # 请求间隔3-6秒
    'scroll_delay': (1, 2),         # 滚动延时1-2秒
    'page_load_wait': (2, 4),       # 页面加载等待2-4秒
    'max_retry': 3,                 # 最大重试3次
    'jobs_per_page': 30,            # 每页约30条
}
```

### 修改控制参数

如需调整，编辑 `crawler/boss_smart_spider.py` 文件中的 `self.config`

**建议**:
- 首次使用保持默认
- 如触发反爬虫，增加延时
- 如速度太慢，可适当减少延时

## 🔍 查看和管理Cookie

### 查看Cookie

```bash
cat data/cookies.json
```

### 删除Cookie（重新登录）

```bash
rm data/cookies.json
```

下次运行时会要求重新登录

### Cookie失效处理

**现象**: 爬虫提示需要登录

**原因**:
- Cookie过期（通常7-30天）
- Boss直聘更新了验证机制

**解决**:
1. 删除旧Cookie: `rm data/cookies.json`
2. 重新运行爬虫
3. 手动登录一次

## ⚠️ 常见问题

### 1. 检测到验证码

**现象**: 程序暂停，提示"检测到验证码"

**处理**:
1. 切换到浏览器窗口
2. 完成滑块/点击验证
3. 回到终端按回车继续

### 2. 爬取速度慢

**原因**: 防止触发反爬虫，设置了随机延时

**优化**:
- 方案1: 使用`--headless`无头模式
- 方案2: 减少`delay_range`（风险：可能被封）
- 方案3: 分批爬取

### 3. 未获取到数据

**可能原因**:
1. Cookie失效 → 删除Cookie重新登录
2. 页面结构变化 → 更新选择器
3. 被临时封禁 → 等待1小时后重试

### 4. 程序中断

**不用担心**: 已爬取的数据会自动保存

**恢复方法**:
1. 查看保存的数据: `ls data/raw/`
2. 继续爬取剩余的数据
