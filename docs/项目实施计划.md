### ⏳ 阶段9：职位推荐算法

**时间安排**：第8周
**完成状态**：⏳ 待实施

#### 任务清单
- [ ] 文本预处理
  - [ ] 中文分词（jieba）
  - [ ] 停用词过滤
  - [ ] 文本清洗
- [ ] TF-IDF特征提取
  - [ ] 职位描述向量化
  - [ ] 特征权重计算
- [ ] 余弦相似度计算
  - [ ] 职位间相似度矩阵
  - [ ] 相似职位查找
- [ ] 推荐系统实现
  - [ ] 基于内容的推荐
  - [ ] 基于浏览历史的推荐
  - [ ] 推荐结果排序

#### 算法流程
```python
1. 数据预处理
   ↓
2. jieba分词 → 去停用词
   ↓
3. TF-IDF向量化
   ↓
4. 计算余弦相似度
   ↓
5. 推荐TOP-N职位
```

#### 推荐策略
- **冷启动**：基于用户填写的求职意向推荐
- **热启动**：基于用户浏览/收藏历史推荐
- **混合推荐**：结合用户行为和内容相似度

#### 产出物
- `ml_models/recommender.py` - 推荐算法
- `apps/recommend/views.py` - 推荐接口
- `templates/recommend.html` - 推荐页面

  - [ ] 交叉验证
  - [ ] 评估指标（MAE、RMSE、R²）

- [ ] 模型保存与部署
  - [ ] 模型序列化（joblib）

#### 特征设计

**输入特征**：
- 职位类型（Python、Java等）
- 城市（北京、上海等）
- 学历（本科、硕士等）
- 工作经验（1-3年、3-5年等）
- 公司规模（可选）

**输出**：
- 回归：预测具体薪资值
- 分类：预测薪资区间（5-10K、10-15K等）

#### 模型对比

| 模型 | 优点 | 缺点 | 应用场景 |
|-----|------|------|---------|
| 随机森林 | 准确率高、抗过拟合 | 计算慢 | 薪资具体值预测 |
| 逻辑回归 | 速度快、可解释性强 | 线性限制 | 薪资区间分类 |

#### 产出物
- `ml_models/predictor.py` - 预测模型
- `data/models/salary_model.pkl` - 训练好的模型
- `apps/predict/views.py` - 预测接口
- `templates/predict.html` - 预测页面
